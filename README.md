# 💖 愿此朝夕陪伴你的生活

<div align="center">

**在流动的朝夕里，伴着一日光影、生活节律、心绪起落、关系记忆与聊天余温，自主安排作息、穿搭、出行与回应；让角色不止等待消息，而是拥有自己的晨昏与心事，也会在恰好的片刻想起你、靠近你，陪你慢慢经过平凡生活。**


</div>

---

## 📖 这是什么

这是一个给角色补上“日常生活背景”的插件，Ta不再只是被叫到才出现的回复者。

清晨会随着今天的主题醒来，看看天气，想想要不要出门；午后可能摸鱼、观察群聊、补一点能量；夜深时会根据疲惫、睡眠债和打扰程度，慢慢进入浅休息、浅睡眠或深度睡眠。

这不是一张固定日程表，而是一套会呼吸的生活系统。Ta有自己的心情、穿搭、关系印象、地点记忆、生活事件和一点点主观小心思。聊天发生时，这些会作为隐藏生活背景参与判断，让回应更像从Ta正在过的日子里自然长出来。

## 🌟 核心能力

### 1. 🕰️ 朝夕流转：自主生活日程

* **每日生活背景**：生成主题、心情色彩、日程类型、日程基调、日程倾向、天气、穿搭、时间轴和当天小事件。
* **业务日边界**：以每日生成时间作为新一天边界，凌晨不会硬切到空白状态。
* **周计划延续**：每周生成主题、目标、每日提示和建议活动，也支持自定义周模板。
* **邀约打断**：用户提出邀约时，智能判断接受、拒绝、改约或合并进后续日程。

### 2. 🧠 心绪小雷达：状态与注意力

* **状态维度**：体力、心情、忙碌度、社交意愿、无聊值、摸鱼值、睡眠质量、观看状态和注意力开放度会持续变化。
* **消息打断等级**：不是每条消息都强行打断生活，而是结合当前状态、关系和消息重要性判断是否值得关注。
* **看见与没看见**：没看见不影响关系；看见但不理会留下主观痕迹，后续可能影响态度、回想或回避。
* **状态变化记录**：保存为什么想看群、为什么没回、为什么被打断、为什么切换成观察状态等内部裁定。

### 3. 👗 今日穿搭：不是换装按钮，是生活判断

* **穿搭适配判断**：外出、休息、天气变化、日程变化时，智能判断当前穿搭是否合适。
* **保持也算决定**：不换装不是偷懒，而是当前穿搭仍适合当下生活。
* **分时段历史**：晨间、白天、夜晚的穿搭变化会被记录，方便回看一天里Ta怎么整理自己。
* **素材只是灵感**：主题、心情色彩、穿搭风格、日程类型和发型素材不会硬性抽签，最终仍由模型判断。

### 4. 💬 聊天余温：关系、记忆与主动靠近

* **关系档案**：沉淀互动次数、近期印象、主观称呼、短标签、关系叙事和来源会话。
* **心理新鲜度**：消息不会只按时间过期，会根据相关度、情绪、关系和当前兴趣衰减；后续同一话题可以重新激活。
* **主动回复**：会话安静一段时间后，结合生活状态、注意力开放度、关系印象和聊天氛围判断要不要自然续一句。
* **私聊回访**：可选开启。Ta会在合适的时候根据关系记忆和生活状态判断是否主动想起某个人。
* **不抢正常聊天**：主动回复只在沉默后触发，不拦截命令，也不替代原本聊天流程。

### 5. 🌙 夜晚收束：睡眠、复盘与偏好学习

* **睡眠分层**：浅休息、浅睡眠、深度睡眠由能量、时间、昨日睡眠债和打断等级共同影响。
* **每日复盘**：夜间或次日生成前沉淀当天总结、记忆点、偏好点和可延续生活事件。
* **偏好学习**：从聊天、复盘、状态演化和邀约结果中学习稳定偏好，再影响之后的日程。
* **生活事件**：保存会延续几天的小事，让后续生活不再每天从零开始。

### 6. 🗃️ 记忆小抽屉：结构化数据

* **每日记录**：日程、时间轴、天气、穿搭、状态和当天事件。
* **状态日志**：体力、心情、社交意愿、睡眠和注意力变化。
* **世界记忆**：关系档案、地点档案、事件记忆、聊天摘要、群聊氛围和消息留意。
* **计划约定**：周计划、自定义周模板、备忘录、邀约和未来承诺。
* **自动整理**：按保留策略定期整理旧记录，让记忆保持清爽。

### 7. 🎙️ 声音、影像与表情：把生活说出来、拍下来

* **Gemini 原生生图**：支持文生图和图生图，能把当前生活状态、穿搭、场景和角色形象参考图整理成更适合出图的提示词。
* **角色形象参考**：可在面板上传并管理多张角色参考图；需要时会提供给模型，用于保持角色外貌、发型和整体气质一致。
* **Grok 视频生成**：支持把日常片段整理成短视频提示词，后台等待生成完成后再发送，不拖住正常聊天。
* **SiliconFlow 情绪语音**：可让角色把一句话直接说出来，失败时才回落为文字，不会强行附带一段重复文本。
* **语音智能切换**：由模型先判断“这句适不适合说出口”，再结合节奏门控、黑白名单和最近发送情况决定是否发语音。
* **表情与动作意图**：记忆提炼时会记录情绪、动作和表情倾向，让后续表达更像角色自然反应，而不是只会打字。

## 🚀 快速开始

1. **安装并启用**
   - 在插件市场安装 `astrbot_plugin_daily_life`。
   - 启用后会自动创建全新的结构化数据。

2. **设置生成时间**
   - 默认是 `07:00`。
   - 可发送 `/生活 时间 07:30` 修改。
   - 这个时间既是每日生成时间，也是业务日边界。

3. **选择模型**
   - 生成模型负责完整日程、周计划和未单独指定的任务。
   - 穿搭模型、状态模型、复盘模型、邀约模型、素材模型、记忆模型、主动回复模型都可以单独指定。
   - 指定模型失败时会临时使用当前默认模型继续；指定模型恢复后会自动回到指定模型。

4. **配置天气和称呼**
   - 默认城市会影响天气、日程和穿搭。
   - 如果平台无法稳定获取昵称，可在本地用户称呼映射里补充 `用户ID:称呼`，它会用于聊天记忆、邀约、备忘录、参考私聊、MemOS 和面板展示。

5. **生成第一天**
   - `/生活 重置`：生成当前业务日完整生活背景。
   - `/生活 显示`：查看今天安排。
   - `/生活 状态`：查看Ta此刻正在做什么。

6. **打开仪表盘**
   - 在网页管理后台的插件详情页进入 `dashboard`。
   - 可查看状态、世界记忆、生活复盘、素材工坊，也可以直接调整设置。

## 💬 自然语言交互

开启工具调用后，很多事情不用死记指令，也可以直接自然地说。

| 你的说法 | Ta会尝试做什么 |
| :--- | :--- |
| “下午一起去看电影吧” | 判断邀约是否能打断今天的计划，能否接受、改约或合并进后续日程 |
| “明天记得买牛奶” | 写入明天备忘录，让明日生活生成时自然参考 |
| “你现在在忙吗” | 结合当前活动、体力、心情和社交意愿回答 |
| “今天适合出门吗” | 结合天气、日程、地点倾向和当前状态回答 |
| “你这身适合出去玩吗” | 结合当前穿搭、天气和外出安排判断是否合适 |
| “今晚早点睡吧” | 作为聊天记忆、承诺或生活倾向被参考，不需要硬性的睡觉指令 |

自然语言只提供生活线索，最终仍由模型根据角色状态和当天背景自主判断。

## 🌐 联网灵感

周计划、周模板、素材和发型工坊支持联网填充或联网新建。

它复用网页搜索配置中的 Tavily / Brave 能力。开启后，你可以输入“少女春日穿搭”“宅家治愈周”“夜间发型灵感”这类提示词，让搜索结果先提供现实参考，再由模型整理成符合当前角色风格的素材。

联网结果不是硬规则，只是给生活生成添一点新鲜空气。最终是否采纳、怎样融入，仍由模型结合当天状态和角色设定判断。

## 🧠 MemOS 外部长期记忆

插件可以连接 MemOS 托管服务，把外部长期事实、偏好和纠错作为隐藏参考注入聊天。

默认是**只检索，不写入**：聊天前短暂等待 MemOS 返回相关记忆，超时或失败就跳过，不影响正常回复。开启精选同步后，只会把本插件已经判断值得长期复用的摘要、偏好、关系点、备忘录和记忆纠错异步同步过去，不会把每一轮聊天都塞进外部记忆。

内置记忆更像Ta自己的生活小抽屉，负责当天状态、关系印象、复盘和世界记忆；MemOS 更像可选的外部长记忆书架，适合和其他记忆系统共享稳定事实。

| `web_inspiration_config` | 联网灵感搜索和各工坊搜索提示词 |
| `storage_config` | 每日记录、复盘、世界记忆、计划约定和素材的保留策略 |
| `weekly_theme_config` | 周计划生成日、生成时间、默认模板和内置模板权重 |
| `story_engine_config` | 日程生成提示词片段，包含穿搭、状态、时间轴、地点事件和聊天人物规则 |

<details>
<summary><b>几个容易忽略的小设置</b></summary>

1. **生成时间不是普通闹钟**
   - 它同时决定每日生活背景生成时间和业务日边界。
   - 例如设置为 `07:00`，那么凌晨 `02:00` 仍会被视作前一天的深夜延续。

2. **穿搭模型只负责实时判断**
   - 完整日程生成里的穿搭跟随生成模型。
   - 自动生活状态更新、手动重置保持、外出适配判断等实时场景会使用穿搭模型。

3. **主动回复默认克制**
   - 它不会在普通聊天时立刻抢话。
   - 只有会话安静超过配置时间，且通过模型裁定，才可能自然续一句。

4. **私聊回访建议谨慎开启**
   - 开启后会根据关系档案、近期记忆和生活状态判断是否主动私聊。
   - 适合强陪伴场景，不适合只想安静记录生活背景的场景。

5. **素材不是硬规则**
   - 心情色彩、日程类型、穿搭风格和发型素材只是灵感库。
   - 最终仍由模型结合当天生活自主判断，不会强制抽签。

6. **没有日程时不会硬编**
   - 当前业务日没有生活记录时，会注入防编造隐藏约束。
   - 需要完整生活背景时，可用 `/生活 重置` 或在面板里重生成今日。

</details>

## 🧯 常见问题

### Q1：为什么不用 `/生活 睡觉`、`/生活 熬夜`？

睡觉和熬夜不应该是硬按钮。现在会由模型结合时间、能量、睡眠债、当前日程和消息打断等级，自主判断浅休息、浅睡眠、深度睡眠或熬夜状态。

### Q2：为什么外出也可能不换装？

外出会触发穿搭适配判断，但结果不一定是换装。如果当前穿搭已经适合天气、地点和日程，模型可以选择保持。

### Q3：主动回复会不会抢走正常聊天？

不会。主动回复只记录普通会话活跃度，并在会话沉默后后台裁定是否自然续一句。命令和正常聊天不会被它替代。

### Q4：为什么有些消息“看见但没回”？

这是拟真机制的一部分。角色可能看见了，但因为正在忙、注意力不开放、关系不够近、话题不合适或当下没有回应欲，所以选择观察或不接话。

### Q5：指定模型失败后会一直降级吗？

不会长期锁死。某次指定模型失败时，会回到当前默认模型继续完成当前任务；后续指定模型恢复后会自动继续使用指定模型。

### Q6：群成员能私聊回访吗？

普通群成员不一定是好友，直接私聊可能发送失败。插件会记录关系来源和群来源，私聊回访更适合已建立私聊上下文或好友关系的对象。

### Q7：个人微信为什么建议配置本地称呼？

个人微信接口不一定稳定提供真实昵称。为了让聊天记忆、邀约、备忘录、私聊参考、MemOS 和面板展示更自然，建议用本地用户称呼映射补充。它只补充显示称呼，不会替代人设线索、主观称呼或性别关系校准。

### Q8：清空数据会删掉什么？

`/生活 清空` 会清空全部日常生活背景数据。若只想整理部分数据，优先使用 `/生活 存储` 查看分类，再按分类清理或清空。

### Q9：语音智能切换为什么有时还是发文字？

它不是“开启后每句都变语音”。模型会先判断这句话是否适合说出口，再经过节奏门控：太频繁、黑名单限制、当前会话不适合、刚发过语音、内容更适合文字留存，都会改为文字。用户明确要求“发句语音”时，会优先按语音请求处理。

### Q10：图片、视频或语音生成失败会怎样？

不会卡住正常聊天。图片和视频工具会返回失败原因；语音在主动回复场景里会优先尝试直接发语音，生成或发送失败时再兜底为文字。

### Q11：MemOS 和内置记忆有什么区别？

内置记忆负责本插件自己的生活状态、关系印象、复盘、偏好和世界记忆；MemOS 是可选的外部长期记忆服务。开启后聊天前会短超时检索外部记忆，精选同步也只同步值得长期复用的内容。

### Q12：角色形象参考图有什么用？

图片生成会把下方角色形象参考图提供给模型，用于保持角色外貌一致。默认策略是“不使用”，你可以在面板里上传、删除和排序多张参考图，再按需要改成按场景判断或始终参考。

## 🌰 生活片段示例

**场景 A：出门前的小判断**

> 生活背景：午后有外出安排，天气微热。  
> Ta的判断：当前穿搭是轻便休闲风，适合短途出门，只需要把头发重新扎好，不必整套换装。  
> 后续影响：面板记录“保持当前穿搭，轻微整理发型”，聊天时也会自然带出“正准备出门”的生活感。

**场景 B：群聊里的安静观察**

> 生活背景：Ta正在看综艺，注意力半开放。  
> 群聊消息：群里有人聊到一个Ta有点熟悉的话题。  
> Ta的判断：看见了，但当前话题已经有人接得很好，Ta暂时不插话，只留下“扫过但未回应”的主观痕迹。  
> 后续影响：几分钟后如果有人继续这个话题，之前扫过的内容可能被重新激活。

**场景 C：夜里慢慢睡去**

> 生活背景：今天社交消耗偏高，睡眠债也有一点。  
> 消息打断：普通消息不足以叫醒Ta，强相关消息可能让Ta短暂浅醒。  
> Ta的状态：从浅休息进入浅睡眠，再慢慢沉到深度睡眠。  
> 后续影响：第二天体力、心情和日程倾向会自然接续，而不是重新从满格开始。
